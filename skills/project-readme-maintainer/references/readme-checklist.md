# README Checklist

Use this checklist before finalizing documentation updates.

## Root README
- Explain what the project solves and who should use it.
- Document prerequisites and installation steps.
- Document end-to-end run flow.
- Link each important subdirectory README.
- Include data/model artifact policy and storage expectations.
- Include troubleshooting and FAQ.

## Subfolder README
- Explain scope of this folder.
- List key files and their roles.
- Show local commands with concrete examples.
- State inputs/outputs and where artifacts are written.
- Link back to root README.

## Quality Gates
- Commands are copy-paste runnable.
- Paths and filenames are accurate.
- Links resolve correctly from markdown preview.
- No stale instructions contradict repository state.
